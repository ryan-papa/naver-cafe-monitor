## 지적 사항

### [P1] 실패 감지 책임의 잘못된 위임
- **위치:** F-27
- **원문(수정 전):** "연속 실패 알림은 선행 PRD의 배치 401 알림 경로가 담당"
- **지적:** 선행 PRD(F-17/F-18)의 알림은 `refresh_token` 만료 14일 전 경고 전용이며, 배치 401 실패 알림은 존재하지 않음. 따라서 refresh cron이 수시간~수일 조용히 실패하고 배치의 401 자동갱신이 access token을 지속 발급하는 구간에서는 운영자가 어떤 신호도 받지 못함. 핵심 목표(refresh token 장기 rotation 보장)가 운영적으로 비어 있는 상태.

## 반영

- **F-27 수정:** 잘못된 위임 문구 제거, 최종 안전망으로 선행 PRD F-17/F-18의 만료 14일 전 경고가 최악 시나리오를 포착함을 명시
- **수용 리스크 섹션 추가:** cron 장시간 실패 시 운영자 신호 지연(최대 ~14일)을 명시적으로 수용 리스크로 문서화
- **Out of Scope 보강:** 즉시/연속 실패 알림 제외 근거를 수용 리스크 항목으로 링크
