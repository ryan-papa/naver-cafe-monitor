# PRD — 대시보드 페이지네이션 + 게시물 상세 모달

| 항목 | 값 |
|------|-----|
| 프로젝트 | naver-cafe-monitor |
| 작성일 | 2026-04-17 |
| 브랜치 | `feat/dashboard-pagination-detail-modal` |
| PRD ID | ce169293 |

## 1. 프로젝트 개요

| 항목 | 내용 |
|------|------|
| 목적 | 대시보드를 게시판형 UI로 개선 — 페이지네이션으로 과거 기록 탐색, 상세 모달로 게시물·전송 내용 확인 |
| 대상 사용자 | 관리자 (세화유치원 공지 모니터링 담당) |
| 배경 | 현재 대시보드는 전체 목록을 한번에 노출 → 데이터 누적 시 탐색 불편, 카톡 전송 내용 확인 불가 |

## 2. 핵심 시나리오 + 실패 모드

| # | 시나리오 | 코드경로 | 감지 | UX 대응 |
|---|---------|----------|------|---------|
| S1 | 과거 게시물 탐색 | `GET /api/posts?offset=&limit=` → 프론트 페이지 계산 | 응답 status, total 필드 | 버튼 비활성·로딩 스피너 |
| S2 | 페이지 크기 변경 | Select onChange → offset=0 리셋 후 재조회 | — | 즉시 재조회 |
| S3 | 게시물 상세 열기 | 행 클릭 → `GET /api/posts/{id}` → 모달 렌더 | 404 → 에러 토스트 | 모달 내 에러 표시 |
| S4 | 모달에서 재발송 | 모달 재발송 버튼 → `POST /api/posts/{id}/resend` | 실패 시 토스트 | 버튼 로딩·비활성 |
| S5 | total=0 빈 상태 | list 응답 items 없음 | items.length === 0 | "데이터 없음" 안내 + 페이지네이션 숨김 |
| S6 | 페이지 범위 초과 | offset ≥ total | API 응답 items=[] | 마지막 페이지로 자동 이동 |

## 3. 대안 탐색

### 3.1 카톡 전송 내용 표시 방식

| 대안 | 장점 | 단점 | 공수 | 근거 유형 |
|------|------|------|------|----------|
| A: DB 필드로 실시간 재구성 ★ | 스키마 변경 無, 과거 데이터 공란 없음 | 실제 전송 텍스트와 미세 차이 가능 | S | 엔지니어 선호 |
| B: `kakao_sent_content` 컬럼 추가 | 실제 전송 텍스트 그대로 보존 | 마이그레이션, 과거 데이터 공란 | M | — |
| C: 별도 전송 로그 테이블 | 분할 전송 이력 완전 추적 | 스키마 확장·조인 비용 | L | — |

**선택: A** — 과거 데이터 유실 없이 최소 공수로 요구사항 충족.

### 3.2 모달 UX

| 대안 | 장점 | 단점 | 근거 유형 |
|------|------|------|----------|
| A: 읽기 전용 | 단순 | 재발송 동선 분리 | — |
| B: 본문+카톡+재발송(모달 내 이동, 행 버튼 제거) ★ | 동선 일원화, UI 군더더기 제거 | 행에서 바로 재발송 불가 | 제품 가설 |
| C: 모달+행 버튼 병행 | 빠른 접근 | UI 중복 | — |
| D: 본문·카톡 탭 분리 | 정보 구역 분리 | 클릭 단계 증가 | — |

**선택: B**

## 5. 기능 요구사항

| ID | 기능 | 우선순위 | 수용 기준 |
|----|------|:--------:|-----------|
| F-01 | 페이지네이션 UI (맨앞/앞/번호/뒤/맨뒤) | P0 | 5개 버튼 + 현재 페이지 기준 ±2 번호 노출, 경계 시 비활성 |
| F-02 | 페이지 크기 Select (10/30/50, 기본 30) | P0 | 변경 시 offset=0 리셋 후 즉시 재조회 |
| F-03 | 게시물 행 클릭 → 상세 모달 오픈 | P0 | `GET /api/posts/{id}` 호출, 로딩·에러 상태 처리 |
| F-04 | 모달: 본문 영역 (제목·원본 URL·날짜·요약) | P0 | DB 필드 그대로 표시, URL은 링크 |
| F-05 | 모달: 카카오톡 전송 내용 재구성 표시 | P0 | `send_notice_summary` 포맷 모사 (`[세화유치원 공지]\n\n📋 {title}\n\n{summary_content}`), `[일정 정리]` 분리 시 2개 블록 |
| F-06 | 모달: 재발송 버튼 (기존 리스트 행 버튼 제거) | P0 | `POST /api/posts/{id}/resend` 호출, SUCCESS 상태만 활성 |
| F-07 | 모달: ESC·배경 클릭·닫기 버튼으로 닫기 | P0 | 포커스 트랩 + 키보드 접근성 |
| F-08 | URL 상태 동기화 (`?page=N&size=S`) | P1 | 새로고침·뒤로가기 시 상태 복원 |
| F-09 | 빈 상태·범위 초과 처리 | P0 | items=0 시 안내, offset≥total 시 마지막 페이지 이동 |

## 7. 기술 스택

| 레이어 | 기술 |
|--------|------|
| 프론트 | Astro + 바닐라 TS (기존 `web/src/pages/index.astro` 패턴 유지) |
| API | FastAPI `/api/posts`, `/api/posts/{id}`, `/api/posts/{id}/resend` (기존 활용, 변경 없음) |
| DB | MySQL `posts` 테이블 (스키마 변경 없음) |

## 8. 제약사항

| 분류 | 내용 |
|------|------|
| 성능 | limit ≤ 100 (기존 API 제약 유지), 페이지 전환 응답 < 300ms |
| 보안 | 관리자 페이지 동일 접근 제어 (변경 없음), XSS 방어: 모달 본문 렌더 시 textContent 사용 |
| 호환성 | 최신 Chrome/Safari 기준, 모바일 반응형 유지 |
| 접근성 | 모달 포커스 트랩, ESC 닫기, 버튼 `aria-label` |
