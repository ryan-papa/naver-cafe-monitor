# PRD: 대시보드 디자인 개선 + 카톡 재발송

**ID:** 20260416_150000_dashboard-redesign-resend_a4f8k2m7
**날짜:** 2026-04-16
**상태:** Approved

---

## 문제·맥락

- **배경:** 현재 대시보드가 Bootstrap 기본 스타일에 가까워 시각적 완성도가 부족. 처리된 게시글 히스토리를 확인만 할 수 있고, 카톡 알림을 다시 보낼 수 없음
- **원인:** 초기 MVP로 기능 우선 구현. 재발송 요구사항이 당시에 없었음
- **왜 지금:** UPA 스타일의 모노크롬 미니멀 디자인 참고 레퍼런스 확보. 재발송 니즈 발생 (놓친 알림 재확인)

## 타깃 사용자와 사용 맥락

| 항목 | 내용 |
|------|------|
| 대상 사용자 | 관리자 (카페 모니터링 시스템 운영자) |
| 사용 맥락 | 대시보드에서 처리 이력 확인 중, 특정 게시글의 카톡 알림을 다시 받고 싶을 때 |
| 핵심 동기 | 놓친 알림 재수신, 다른 기기에서 재확인 |

## 핵심 시나리오

| # | 장면 | 사용자 행동 | 시스템 반응 |
|---|------|-----------|-----------|
| 1 | 대시보드 접속 | 페이지 로드 | 개선된 모노크롬 UI로 히스토리 표시 |
| 2 | 히스토리 확인 | 필터/정렬로 게시글 탐색 | 기존과 동일한 필터링 동작 |
| 3 | 재발송 클릭 | SUCCESS 상태 행의 "재발송" 버튼 클릭 | 버튼 로딩 → API 호출 → 카톡 발송 |
| 4 | 결과 확인 | 대기 | 토스트로 "발송 완료" / "발송 실패" 표시, 버튼 30초 쿨다운 |

**상태 분기:**

| 상태 | 대응 |
|------|------|
| 빈 상태 | 기존과 동일 (데이터 없음 안내) |
| 에러 | 토스트로 실패 메시지 표시 |
| 로딩 | 버튼 스피너로 발송 중 표시 |
| 쿨다운 | 버튼 비활성화 + 남은 시간 표시 (30초) |
| summary 없음 | 재발송 버튼 비활성화 (발송 내용 없음) |

## 스코프 & 비-스코프

| 포함 (In) | 비포함 (Out) |
|-----------|-------------|
| 대시보드 비주얼 개선 (모노크롬 베이스 + 뱃지 컬러) | 레이아웃 구조 변경 (테이블 → 카드 전환 등) |
| 행별 재발송 버튼 (SUCCESS만) | 체크박스 일괄 선택 |
| `POST /api/posts/{id}/resend` API | 이미지 재발송 (텍스트 summary만) |
| 토스트 알림 피드백 | summary 편집 기능 |
| 30초 쿨다운 | 일일 횟수 제한 |

## 성공 기준 & 실패 기준

| 구분 | 지표 | 목표값 |
|------|------|--------|
| 성공 | 재발송 API 호출 → 카톡 수신 | 정상 동작 |
| 성공 | 디자인 개선 후 기존 기능 정상 | 필터/정렬/페이징 동작 유지 |
| 실패 | 재발송 시 카톡 미수신 | API 200이나 카톡 안 옴 |

## 가정 · 리스크 · 열린 질문

| 유형 | 내용 | 검증 방법 |
|------|------|----------|
| 가정 | API 서버에서 batch의 KakaoMessenger를 import 가능 | sys.path 확인 |
| 가정 | 카카오 토큰이 API 서버 환경에서도 접근 가능 | config 경로 확인 |
| 리스크 | API 프로세스에서 카카오 토큰 갱신 시 batch와 충돌 | 토큰 파일 atomic write로 완화 (이미 구현됨) |
| 리스크 | 카카오 API rate limit | 30초 쿨다운으로 완화 |
