## 개요·목적

- **배경:** 네이버 카페 특정 게시판을 수동으로 반복 확인하는 번거로움을 자동화
- **목표:** 새 게시물 자동 감지 → 게시판 유형별 처리(이미지 업로드 / AI 요약) → 카카오톡 알림 전달
- **범위:** 지정 카페·게시판 크롤링, 이미지 Google Photos 보관, 공지 AI 요약, 카카오톡 메시징, cron 스케줄러
- **대상 카페:** 세화유치원 (https://cafe.naver.com/sewhakinder, cafe_id: 31672965)

---

## 대상 게시판 및 처리 플로우

### 사진 게시판 (menus/13 — 지혜반 이야기)

1. 새 게시물 감지 (`last_seen.json` 기반, post_id 비교)
2. 전체 이미지 다운로드 (얼굴 인식 필터 비활성, 전체 다운로드 정책)
3. Google Photos "26년 세화유치원" 앨범에 업로드
4. 게시글 본문 텍스트 → Claude CLI sonnet 모델로 전달사항 요약
5. 카카오톡 전송: 전달사항 메시지 + 사진 업로드 알림 메시지 (별도 전송)
6. 버튼: "카페에서 보기" → 모바일 카페 링크 (`m.cafe.naver.com/sewhakinder/{pid}`)

### 공지사항 (menus/6 — 세화소식)

1. 새 게시물 감지 (`last_seen.json` 기반)
2. 공지 이미지 다운로드
3. Google Photos "26년 세화유치원 공지" 앨범에 업로드
4. 이미지 2등분 + 3등분 병렬 분석 (Claude CLI opus 모델, 경계 잘림 방지)
5. 통합 요약 (7세/전체 대상만 포함, 5세/6세 한정 내용 제외)
6. 카카오톡 전송: 전체 내용 메시지 + 일정 정리 메시지 (별도, `[일정 정리]` 구분자로 분리)
7. 버튼: "카페에서 원문 보기" → 모바일 카페 링크
8. 2000자 초과 시 줄바꿈 기준 자동 분할 전송

## 인증 방식

| 서비스 | 방식 |
|--------|------|
| 네이버 카페 | 수동 로그인(`python -m src.crawler.login`) → 쿠키 저장(`data/cookies.json`) → 이후 쿠키 복원 (2차 인증 대응) |
| 카카오톡 | REST API 직접 호출 (PyKakao 미사용), OAuth access_token |
| Google Photos | OAuth + refresh_token 자동 갱신 (`config/google_token.json`) |
| AI 요약 | Claude Code CLI (별도 API 키 불필요, 로컬 인증 사용) |
