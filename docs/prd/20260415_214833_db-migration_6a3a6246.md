**상태:** Approved

## 개요·목적

- **배경:** 현재 게시글 처리 이력을 `last_seen.json` 파일로 관리하여 최신 게시글 ID만 보관. 처리 이력 조회, 성공/실패 추적, 화면 기반 모니터링이 불가능한 상태.
- **목표:** 게시글 처리 이력을 MySQL DB로 이관하고, FastAPI 백엔드 + Astro 프론트엔드를 추가하여 처리 내역을 화면에서 조회 가능하게 함.
- **범위:** batch(기존) + api(신규) + web(신규) 3-Tier 모노레포 구성. 토큰/쿠키/얼굴 인코딩 등 기타 파일 저장소는 현행 유지.

## 사용자 스토리

| As a | I want | So that |
|------|--------|---------|
| 운영자 | 웹에서 게시글 처리 이력을 목록으로 조회하고 싶다 | 정상 처리 여부를 한눈에 파악할 수 있다 |
| 운영자 | 게시판별, 상태별로 필터링하고 싶다 | 실패 건만 빠르게 확인할 수 있다 |
| 운영자 | 처리 일시 기준으로 정렬하고 싶다 | 최근 작업부터 확인할 수 있다 |
| 운영자 | batch 실행 시 처리 결과가 DB에 자동 기록되길 원한다 | 별도 관리 없이 이력이 누적된다 |

## 테이블 설계

```sql
CREATE TABLE posts (
    id           BIGINT AUTO_INCREMENT PRIMARY KEY,
    reg_ts       TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP,
    upd_ts       TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    board_id     VARCHAR(50) NOT NULL COMMENT '게시판 식별자 (menus/6, menus/13)',
    post_id      BIGINT NOT NULL COMMENT '게시글 번호',
    title        VARCHAR(500) NOT NULL DEFAULT '' COMMENT '게시글 제목',
    summary      TEXT COMMENT '요약 내용',
    image_count  INT NOT NULL DEFAULT 0 COMMENT '이미지 수',
    post_date    DATETIME COMMENT '게시글 원본 업로드 일시',
    status       ENUM('SUCCESS', 'FAIL') NOT NULL DEFAULT 'SUCCESS' COMMENT '처리 결과',
    UNIQUE KEY uk_board_post (board_id, post_id),
    INDEX idx_status (status),
    INDEX idx_reg_ts (reg_ts)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 제약사항

- 기술: DB 접속 시 SSL 클라이언트 인증서 필수 (eepp.shop 정책)
- 기술: 토큰/쿠키/얼굴 인코딩은 파일 기반 유지 (이번 범위 제외)
- 비즈니스: 기존 batch 동작에 영향 없어야 함 (DB 전환 후에도 동일한 크롤링·발송 동작 보장)
