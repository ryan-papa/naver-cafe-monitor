## 피드백
- 4.2 게시물 상세 UX는 대안이 A/B 2개뿐이라 "인라인 확장 패널" 같은 3번째 대안을 추가하면 대안 탐색 완성도가 더 높아짐.
- S2의 "재발송 성공 시 쿨다운 30초"와 실패 모드 "재발송 실패 시 쿨다운 미적용" 관계를 한 문장으로 명시하면 일관성 혼동 제거(예: "성공 시에만 30s 쿨다운 적용, 실패는 즉시 재시도 가능").
- 내부 시범 사용 게이트를 "배포 후 운영자 1명 × 2일 실사용 + 이슈 0건 확인" 같이 정량화하면 사용자 검증 게이트 점수 상승 가능.
- 성공 기준 중 "육안 검증 통과"는 체크리스트화(로고·사이드바·테이블·버튼·모달 5개 항목 스크린샷 대조)하면 측정가능성 향상.
- 가정 "NanumSquare CDN 접근 가능"은 실패 시 대체 경로(로컬 fallback·system font) 한 줄 추가하면 리스크 완화.
- F-18(다크모드 토큰 이식, Could)은 이번 범위에서 토글 UI 없이 토큰만 이식하므로 사용자 검증 대상 아님을 스코프 비고로 명시하면 혼동 방지.
